# Section Micro-Docs

## Purpose

These files are short implementation notes for agents. They are not product specs.

Read the file that matches the code area you are editing. Update the matching micro-doc when you change a section-level invariant, main entry point, or workflow.

## Invariants

Keep these files short.
Section docs are implementation aids, not product or backend source of truth.

## Agent notes
